Permission Modes

All tools are sandboxed to the project directory. Modes bundle a per-tool permission policy that you can swap from the chat badge or with /mode (Shift+Tab cycles to the next mode). The two built-in modes — Default (mutating tools ask) and Accept Edits (everything auto-allowed) — are editable; use Restore defaults to roll back. Add custom modes by cloning an existing one. The order in the dropdown is the order Shift+Tab cycles through — drag rows to rearrange.

Ignore Patterns

Paths matching these patterns are skipped by file-listing/search tools and refused by file-touching tools. Goal: keep small models from wasting context on caches, deps, and build outputs — not to filter file types. Patterns use .gitignore syntax.
