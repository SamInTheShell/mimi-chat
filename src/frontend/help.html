Guide

Provider

Pick where chat completions are served from. Mimi talks to any OpenAI-compatible /v1/chat/completions endpoint.

Built-in shortcuts

Ollama
Local Ollama daemon at http://localhost:11434/v1. Run ollama serve and pull a model with ollama pull <name>. No API key needed.
LM Studio
LM Studio's local server at http://localhost:1234/v1. Start the server from LM Studio's Local Server tab and load a model there first.
OpenAI compatible
Anything else that speaks the OpenAI chat-completions wire format — llama.cpp's server, vLLM, a remote provider, etc. Paste the base URL ending in /v1.

Endpoint & model

The API Endpoint field is editable for every provider — the cards just preset it. Refresh calls the server's /v1/models to populate the model list. If your server requires it, set the API key as an environment variable on the server side; Mimi sends an empty key by default.

System prompt

Pick a starting prompt — built-in templates ship under Settings → System Prompts, and your custom prompts appear in the same dropdown. You can change the prompt later from the chat composer too.

Project directory

Required. This is the sandbox root for every built-in tool — file reads, edits, listings, and the embedded terminal all run inside this directory. Use Recent to jump back to a previous project, or Browse… for a native folder picker.

Starting a session

Click START SESSION or hit Ctrl/Cmd+Enter. Once in chat, the cwd badge in the header opens a directory switcher and the model badge swaps the active model — to change provider or endpoint, restart Mimi.

Chat

The composer is the home base. Most features are one keystroke or one badge away.

Composer basics

  • Send: Enter. Newline: Shift+Enter.
  • Focus from anywhere: Ctrl/Cmd+I.
  • Stop a reply mid-stream: the stop button or Esc.

Images

Mimi supports image input on vision models. Three ways to attach:

  • Drag & drop image files straight into the composer — no need to aim for a tiny target, anywhere on the input is fine.
  • Paste from the clipboard with Ctrl/Cmd+V (e.g. after a screenshot).
  • Each attached image becomes a draggable pill above the text. Reorder by dragging; remove with the × on the pill.

Extended thinking

The thinking badge toggles whether the request asks the model for its hidden reasoning track. When on, thinking blocks render as collapsible cards above the reply. Toggle from the badge, with Ctrl/Cmd+T, or the /think slash command.

Permission modes

The mode badge in the input bar is the tool-permission policy in effect. Mimi ships with two built-in modes — Default (read-only tools allowed, mutating tools ask) and Accept Edits (everything auto-allowed, still sandboxed to the project directory). You can edit either, clone them, or add your own under Settings → Tools.

  • Click the badge to cycle to the next mode (wraps around).
  • Shift+Tab does the same.
  • /mode (or right-clicking the badge) opens a picker if you'd rather choose by name.
The tool-permission modal
Shown only when the active mode marks the tool as ask. Lists the tool name and a preview of the action. Ctrl/Cmd+Enter allows, Esc denies. The deny-reason field (Alt+Esc to focus) lets you tell the model what to do differently.

System prompts

The prompt badge in the input bar shows the active system prompt. Click it (or Ctrl/Cmd+P, or /system) to switch. Manage the full library under Settings → System Prompts.

Slash commands

Type / at the start of an empty composer for the menu, or just type the command and send.

  • /clear — wipe the conversation context.
  • /think — toggle extended thinking.
  • /mode — open the permission-mode picker.
  • /model — open the model picker.
  • /system — open the system-prompt picker.
  • /save-md — export the conversation as Markdown.
  • /save-json — export the raw conversation + tool payload as JSON (debug).

Terminal snapshots (@)

Type @ in the composer to attach a snapshot of an open terminal tab into the message — useful for asking the model about command output without copy/pasting.

Embedded terminal

Toggle the panel with Ctrl/Cmd+Shift+`. It opens a real PTY rooted in your project directory. Tabs: Ctrl/Cmd+Shift+T new, Ctrl/Cmd+Shift+W close, Ctrl/Cmd+PgUp/PgDn switch, F2 rename.

Scrolling & follow

  • Ctrl/Cmd+J jumps to the bottom and re-engages auto-follow.
  • Ctrl/Cmd+K scrolls to the top of the latest assistant turn.
  • The follow toggle in the scroll cluster controls whether new tokens auto-scroll the view down.

Copy & export

Each message, thinking block, and tool card has a copy button that lifts the raw content to your clipboard. For full-conversation export, use /save-md or /save-json.

Settings

Open from the gear button in any header or with Ctrl/Cmd+,. Same hotkey closes it. Cycle tabs with Ctrl+PgUp/PgDn (or Cmd+Shift+[/] on macOS).

System Prompts

A library of system prompts, stored under ~/.mimi-chat/. The left pane lists prompts; the right pane is the editor.

  • + Add from template seeds a new prompt from one of the built-in role templates (coding agent, writing editor, tutor, brainstorm partner, etc.).
  • + New creates a blank prompt; Clone duplicates the selected one; Delete removes it.
  • The active prompt is set from the chat composer or the setup screen, not here.

Sampling

These fields are merged into every /v1/chat/completions request. Backends that don't support a field usually ignore it.

  • Temperature — randomness of token choice (typical 0–1).
  • Top-p — nucleus sampling mass (0–1).
  • Top-k — limit to the k most likely tokens (llama.cpp-style backends).
  • Min-p — dynamic probability floor relative to the top token.
  • Repeat penalty — discourages repeating tokens (1 = off on most servers).

Restore defaults resets every field at once.

Tools

Two sub-sections.

  • read_file token cap — if a single read would return more than this many estimated tokens, the call is refused so the model re-reads in parts via offset and limit. Default 10000.
  • Permission modes — pick a mode from the dropdown to view or edit its per-tool permissions (Allow, Ask, or Off). Both built-in modes are editable; Restore defaults rolls a built-in back to its shipped settings. Use Clone to seed a custom mode from the current selection, then Rename or Delete to manage it. Cycle modes from the chat with the badge or Shift+Tab.

MCP Servers

The MCP Servers section (under Tools in Settings) lets you connect external tool servers that speak the Model Context Protocol (MCP). Any MCP-compatible server can expose additional tools to the model alongside Mimi's built-in ones.

Adding a server
Click + Add server to create a new entry. Expand the card to configure it:
  • Display name — a label for the sidebar; the server's id (used internally) is derived from this.
  • Command — the executable to launch, e.g. npx, uvx, or an absolute path.
  • Arguments — one per line. For example, -y on one line, then @modelcontextprotocol/server-filesystem, then ~/projects.
  • Environment — optional KEY=VALUE pairs, one per line. Useful for passing API tokens.
  • Working directory — optional; leave empty to inherit the app's working directory.
Lifecycle controls
Each server card shows a status badge (running, starting…, error, or stopped) and three buttons:
  • Start — launches the subprocess and runs MCP handshake + tool discovery.
  • Stop — gracefully shuts down the server process.
  • Refresh — stops then restarts (useful after updating the server or its config).
Toggles
  • Enabled — uncheck to keep the configuration saved but block all tool calls to that server. The model won't see its tools.
  • Auto-start — when checked, the server launches automatically when Mimi starts.
Tool discovery & permissions
Once running, the server's tools appear under the card, namespaced as mcp__<server_id>__<tool_name>. Each tool gets per-mode permissions (Allow, Ask, or Off) — newly discovered tools default to Ask. Edit permissions inline or from the permission-mode editor above.
Debugging
If a server has emitted to stderr, a collapsible Recent stderr section appears at the bottom of its card showing the last few lines of output — helpful for diagnosing startup failures or runtime errors.

Server configurations are persisted in ~/.mimi-chat/mcp.json. Deleting a server (double-click Delete server to confirm) stops the process and removes its tool permissions from all modes.

Ignore

Patterns matching these are skipped by file-listing/search tools and refused by file-touching tools. The goal is to keep small models from wasting context on caches, deps, and build outputs.

  • Honor project .gitignore — when on, the project root's .gitignore is layered on top of the patterns below.
  • Patterns — one per line, .gitignore syntax. Negation (!) and trailing-slash directory rules work like in git. # starts a comment.

DevTools

The DevTools button in the settings header opens Chromium's bundled inspector attached to this window. Ctrl/Cmd+Shift+I works from anywhere in the app.
